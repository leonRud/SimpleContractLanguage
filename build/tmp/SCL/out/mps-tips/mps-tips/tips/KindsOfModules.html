There are four types of modules:

  • Languages defining several aspects of languages that then can be used to write code in solutions
  • Generators that define the conversion rules to transform code into code in another language
  • Solutions holding code (programs or libraries)
  • DevKits bundling multiple languages and solutions together, so that users can express dependencies with a single dependency declaration

Solutions themselves can be of three basic kinds:

  • Sandbox - end-user programs
  • Runtime solutions - libraries of reusable code, frequently loaded from external sources (e.g. jar files)
  • Plugin solutions - code extending the IDE functionality (e.g. new menu entries, actions, tool windows, etc.)
